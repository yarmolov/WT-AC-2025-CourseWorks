# Вариант 34 — Датчики «Умный дом lite» — Роли и действия

Краткое описание

MVP: Ticket, Queue, Agent, Rating, User. UI отображает для каждого типа пользователей свои данные, каждый пользователь делает свои действия. API: /tickets, /queues, /agents, /ratings. Приёмка (MVP): тикеты, статусы, очереди, исполнители, оценки качества

Роли и основные действия

- Администратор

  - CRUD для пользователей
  - CRUD для очередей (Queue)

- Пользователь

  - Создание тикетов (Ticket)
  - Отправка сообщений
  - Оставить отзыв (Rating)

- Агент

  - Принять тикет
  - Закрыть тикет
  - Отправка сообщений

Примеры сценариев

- Пользователь создает тикет, и ждет когда агент нажмет claim, пользователь отправляет сообщения
- Агент принимает тикет, может закрыть тикет и отправлять сообщения
