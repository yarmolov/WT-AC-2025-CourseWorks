# R1 — Роли и действия (ЗОЖ-трекер)

**Коротко:** MVP проекта построен вокруг одного класса пользователей — обычных пользователей, у которых есть собственные метрики и записи. Роли разделять явно не требуется. В дальнейшем можно добавить роль `admin` для управления пользователями и системными задачами.

## Роли

- Пользователь (User)
  - Регистрация / вход
  - CRUD для собственных метрик
  - Создание/редактирование/удаление своих записей (Entry)
  - Просмотр дашборда и отчётов

- Сервис / system (опционально)
  - Импорт/массовая загрузка данных (если появится интеграция)

- Администратор (optional)
  - Управление пользователями
  - Просмотр логов, мониторинг

## Основные действия (map to API)

- POST /api/v1/auth/register — регистрация
- POST /api/v1/auth/login — логин
- GET/POST/PUT/DELETE /api/v1/metrics — управление метриками
- GET/POST/PUT/DELETE /api/v1/entries — управление записями
- GET /api/v1/dashboard — просмотр сводки
- GET /api/v1/reports — получение данных для графиков

## Примеры сценариев

- Пользователь регистрируется → получает дефолтные метрики → заполняет записи по датам.
- Пользователь создаёт новую метрику (например: "Вода") и добавляет дневные значения.
- (Опционально) Администратор просматривает список пользователей и их активность.
