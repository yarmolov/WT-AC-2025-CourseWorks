# R1 — Маршруты / User Flows

1. Аутентификация
   - User: POST `/api/v1/auth/register` → получает `access_token` и стартовые метрики
   - User: POST `/api/v1/auth/login` → получает `access_token`

2. Первичная настройка
   - После регистрации пользователь видит стартовые метрики (Вода/Сон/Шаги)
   - Создаёт дополнительные метрики по потребности

3. Ежедневные операции (User)
   - Добавить запись (Entry) для метрики (быстро с главного экрана)
   - Просмотреть историю и график метрики (страница метрики)
   - Проверить прогресс на дашборде

4. Администрирование (опционально)
   - Просмотр пользователей, логов и системных метрик

5. Экспорт / отчёты
   - GET `/api/v1/reports?metric_id=&date_from=&date_to=` — получить данные для графика/экспорта

6. Ошибки и UX
   - Валидация на вводе (см. `R1_Cards.md`)
   - API возвращает `{ "msg": "..." }` и соответствующий HTTP-код

Приоритеты MVP: регистрация/login, CRUD метрик, добавление записей, дашборд, графики (reports).
