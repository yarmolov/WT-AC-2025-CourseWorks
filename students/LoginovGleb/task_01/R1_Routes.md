# Вариант 40 — Заявки «Да, я в деле» — Маршруты (user flows)

1. Идентификация (вход в систему)

   - Гость: просмотр доступных форм → регистрация/вход → личный кабинет
   - Пользователь: логин → дашборд с заявками и их статусами
   - Модератор: логин → панель рассмотрения заявок
   - Администратор: логин → админ-панель

2. Подготовка данных (Администратор)

   - Заполнить список пользователей
   - Создать формы заявок (Form) с описанием полей
   - Создать статусы (Status) для жизненного цикла заявки
   - Настроить порядок этапов согласования (бонус)

3. Работа с формами (Администратор)

   - Добавить/редактировать/удалить Form
   - Настроить обязательные и опциональные поля формы
   - Привязать возможные статусы к форме

4. Создание заявки (Пользователь)

   - Выбор формы из списка доступных
   - Заполнение полей формы
   - Загрузка вложений (Attachment)
   - Сохранение черновика или отправка заявки

5. Просмотр заявок (Пользователь)

   - Просмотр списка своих заявок
   - Фильтрация по статусу (черновик, на рассмотрении, одобрена, отклонена)
   - Просмотр детальной информации о заявке
   - Отслеживание истории изменений статуса

6. Рассмотрение заявок (Модератор)

   - Просмотр списка заявок на рассмотрение
   - Просмотр деталей заявки и вложений
   - Изменение статуса (одобрить/отклонить/запросить доработку)
   - Добавление комментария к заявке

7. Управление (Администратор)

   - Просмотр всех заявок в системе
   - Управление пользователями и их ролями
   - Просмотр журнала событий
   - Настройка системных параметров

8. Отчёты / История

   - Просмотр журнала событий по заявке
   - Экспорт данных для анализа
   - Статистика по заявкам (по статусам, по периодам)
