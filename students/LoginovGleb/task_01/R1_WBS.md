# Вариант 40 — WBS (эпики → фичи → задачи)

- Эпик A. Модель данных и миграции
  - Фичи: схемы User/Form/Status/Application/Attachment/StatusHistory; миграции
  - Задачи:
    - A1. Описать таблицы и связи (ER-диаграмма)
    - A2. Написать миграции базы данных
    - A3. Подготовить демо-данные (seed)
    - A4. Настроить индексы для производительности

- Эпик B. API и бизнес-логика
  - Фичи: REST API (/applications, /forms, /statuses, /attachments); валидация
  - Задачи:
    - B1. Реализовать CRUD endpoints для Forms
    - B2. Реализовать CRUD endpoints для Statuses
    - B3. Реализовать CRUD endpoints для Applications
    - B4. Реализовать загрузку/скачивание Attachments
    - B5. Добавить пагинацию и фильтры
    - B6. Реализовать валидацию данных по схеме формы

- Эпик C. Трекинг статусов и workflow
  - Фичи: изменение статуса, история переходов, уведомления
  - Задачи:
    - C1. Реализовать механизм смены статуса
    - C2. Создать таблицу истории статусов (audit log)
    - C3. Тесты на корректность переходов статусов
    - C4. Многоэтапное согласование (бонус)

- Эпик D. UI / Интерфейс пользователя
  - Фичи: список заявок, форма создания, трекер статуса, управление вложениями
  - Задачи:
    - D1. Страница списка доступных форм
    - D2. Форма создания/редактирования заявки
    - D3. Компонент загрузки файлов
    - D4. Страница просмотра заявки с историей
    - D5. Дашборд пользователя с заявками
    - D6. Панель модератора для рассмотрения заявок

- Эпик E. Безопасность и роли
  - Фичи: аутентификация, авторизация, RBAC, scope доступа
  - Задачи:
    - E1. Реализовать регистрацию и авторизацию (JWT)
    - E2. Реализовать RBAC (admin, moderator, user)
    - E3. Ограничить доступ по scope (пользователь видит только свои заявки)
    - E4. Тесты доступа для разных ролей
    - E5. Документация прав и ролей

- Эпик F. Окружение и CI/CD
  - Фичи: Docker, Kubernetes manifests (бонус), CI pipeline, тесты
  - Задачи:
    - F1. Создать Dockerfile для backend и frontend
    - F2. Настроить docker-compose для локальной разработки
    - F3. Настроить GitHub Actions/CI pipeline
    - F4. Написать unit-тесты
    - F5. Написать интеграционные тесты
    - F6. Подготовить Kubernetes manifests (бонус)
    - F7. Подготовить инструкции для демо и развёртывания
