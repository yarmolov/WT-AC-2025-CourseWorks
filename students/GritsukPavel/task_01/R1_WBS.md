# Вариант 32 — WBS (эпики → фичи → задачи)

- Эпик A. Модель данных и миграции
  - Фичи: схемы User/Company/Job/Stage/Note/Reminder; миграции
  - Задачи: A1. Описать таблицы и связи; A2. Написать миграции; A3. Подготовить демо‑данные

- Эпик B. API и CRUD операции
  - Фичи: REST API (/jobs, /stages, /notes, /reminders, /companies); валидация
  - Задачи: B1. Реализовать endpoints CRUD для всех сущностей; B2. Добавить пагинацию и фильтры; B3. Тесты API

- Эпик C. Канбан-доска
  - Фичи: отображение вакансий по этапам, drag-and-drop, фильтрация
  - Задачи: C1. Реализовать компонент канбана; C2. Реализовать перетаскивание; C3. Синхронизация с бэкендом

- Эпик D. Напоминания и уведомления
  - Фичи: создание напоминаний, уведомления, отметка выполненных
  - Задачи: D1. CRUD для напоминаний; D2. Система уведомлений; D3. UI для управления напоминаниями

- Эпик E. UI и навигация
  - Фичи: дашборд, формы добавления вакансий, детальные страницы
  - Задачи: E1. Главная страница с канбаном; E2. Формы создания/редактирования; E3. Страницы деталей вакансии

- Эпик F. Безопасность и роли
  - Фичи: аутентификация, авторизация, scope доступа
  - Задачи: F1. Реализовать RBAC; F2. Тесты доступа; F3. Документация прав

- Эпик G. Окружение и CI/CD
  - Фичи: Docker, CI pipeline, тесты
  - Задачи: G1. Dockerfile; G2. Github Actions/CI; G3. Подготовить инструкции для демо
