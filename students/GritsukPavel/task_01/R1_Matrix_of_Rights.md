# Вариант 32 — Матрица прав (операция × роль)

Операции: Просмотр вакансий, Создать/Редактировать/Удалить Job, Управление этапами, Создать/Редактировать/Удалить заметки, Управление напоминаниями, Просмотр канбана, Управление пользователями, Просмотр статистики.

Роли: Администратор, Пользователь

Таблица (кто может что)

- Просмотр вакансий: Администратор (все, read-only), Пользователь (свои, полный доступ)
- Создать Job: Пользователь
- Редактировать/Удалить Job: Пользователь (только свои)
- Управление Company: Пользователь (только свои: CRUD)
- Управление Stage: Пользователь (только для своих вакансий: CRUD)
- Создать/Редактировать/Удалить Note: Пользователь (только для своих вакансий)
- Создать/Редактировать/Удалить Reminder: Пользователь (только для своих вакансий)
- Просмотр канбана: Пользователь (только свои вакансии)
- Перемещение вакансий на канбане: Пользователь (только свои)
- Управление пользователями (CRUD): Администратор
- Просмотр собственного профиля: Пользователь (только self)
- Просмотр общей статистики: Администратор

Примечания:

- По умолчанию пользователь видит только свои вакансии, компании и напоминания (scope ограничение через user_id).
- Администратор имеет доступ ко всем данным в режиме read-only для модерации и аналитики. Редактировать/удалять чужие Job/Company/Note/Reminder администратор НЕ может.
- "Система" — это не роль пользователя, а фоновые процессы (cron/background jobs) для отправки уведомлений о напоминаниях.
- Все операции CRUD требуют проверки ownership: `resource.user_id === req.user.id` (кроме операций администратора).
