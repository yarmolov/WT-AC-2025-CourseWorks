# Вариант 32 — Карточки (обязательные поля и валидации)

1. Форма аутентификации

- username: string, только алфавит и цифры, не пустое → ошибка: «Введите корректное имя пользователя.»
- email: string, формат email, не пустое → ошибка: «Введите корректный email.»
- password: string, минимум 6 символов, не пустое → ошибка: «Пароль должен содержать минимум 6 символов.»

1. Карточка компании (Company)

- id: UUID, автогенерируется
- name: string, не пустое, макс 100 символов → ошибка: «Введите название компании.»
- description: string, опционально, макс 1000 символов
- user_id: reference -> User.id, не пустое → ошибка: «Выберите пользователя.»

1. Карточка вакансии (Job)

- id: UUID
- title: string, не пустое, макс 200 символов → ошибка: «Введите название вакансии.»
- company_id: reference -> Company.id, не пустое → ошибка: «Выберите компанию.»
- user_id: reference -> User.id, не пустое → ошибка: «Выберите пользователя.»
- salary: string, опционально, макс 50 символов
- location: string, опционально, макс 100 символов
- url: string, опционально, формат URL → ошибка: «Введите корректный URL.»

1. Карточка этапа (Stage)

- id: UUID
- job_id: reference -> Job.id, не пустое → ошибка: «Выберите вакансию.»
- name: string, не пустое, макс 50 символов → ошибка: «Введите название этапа.»
- order: integer, не пустое, >= 0 → ошибка: «Укажите порядковый номер этапа.»
- date: datetime, опционально

1. Карточка заметки (Note)

- id: UUID
- job_id: reference -> Job.id, не пустое → ошибка: «Выберите вакансию.»
- content: text, не пустое, макс 5000 символов → ошибка: «Заметка не может быть пустой.»
- created_at: datetime

1. Карточка напоминания (Reminder)

- id: UUID
- job_id: reference -> Job.id, не пустое → ошибка: «Выберите вакансию.»
- title: string, не пустое, макс 200 символов → ошибка: «Введите название напоминания.»
- date: datetime, не пустое, не раньше текущей даты (только при создании) → ошибка: «Дата напоминания должна быть в будущем.»
- completed: boolean, default: false
- created_at: datetime

**Примечание:** Валидация "дата не раньше текущей" применяется только при создании (POST), но не при редактировании (PUT), чтобы можно было обновить прошедшие напоминания.
