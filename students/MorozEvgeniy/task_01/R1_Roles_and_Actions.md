# Вариант 18 — Рецепты «Что приготовить?» — Роли и действия

Краткое описание

MVP: Recipe, Ingredient, Tag, MealPlan, ShoppingItem, User. UI позволяет искать рецепты по ингредиентам, составлять меню на неделю и генерировать список продуктов. API: /recipes, /ingredients, /mealplan, /shopping. Приёмка (MVP): фильтрация по диетам/аллергенам.

Роли и основные действия

- Администратор
  - Управление справочником ингредиентов (Ingredient) — создание, слияние дублей
  - Управление справочником тегов (Tag) — категории, аллергены
  - Управление пользователями (бан/разбан)
  - Модерация рецептов (удаление неприемлемого контента)

- Пользователь (Кулинар)
  - Поиск рецептов по ингредиентам («Что в холодильнике») и тегам
  - Создание и редактирование своих рецептов (Recipe)
  - Добавление рецептов в План питания (MealPlan)
  - Генерация и редактирование Списка покупок (ShoppingItem)
  - Отметка выполненных покупок

- Гость (неавторизованный)
  - Просмотр списка всех рецептов
  - Поиск рецептов и просмотр деталей
  - Регистрация / Вход в систему

Примеры сценариев

- Пользователь открывает приложение, выбирает «Курица» и «Картофель» в фильтре. Система выдает рецепт «Запеченная курица». Пользователь добавляет этот рецепт в План питания на пятницу.
- Пользователь нажимает кнопку «Сформировать список покупок на неделю». Система суммирует все ингредиенты из Плана питания и добавляет их в раздел «Покупки».
- Администратор добавляет новый тег «Кето-диета» и привязывает его к соответствующим ингредиентам или проверяет новые рецепты.
