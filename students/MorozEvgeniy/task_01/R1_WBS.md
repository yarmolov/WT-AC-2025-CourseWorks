# Вариант 18 — WBS (эпики → фичи → задачи)

- Эпик A. Модель данных и миграции
  - Фичи: схемы User, Recipe, Ingredient, MealPlan, ShoppingItem
  - Задачи:
    - A1. ER-диаграмма и описание таблиц
    - A2. Миграции (Knex/TypeORM/SQL)
    - A3. Seed-данные (базовые ингредиенты, теги, тестовые рецепты)
    - A4. Индексы для быстрого поиска по ингредиентам

- Эпик B. API и управление рецептами
  - Фичи: CRUD рецептов, Поиск, Справочники
  - Задачи:
    - B1. API для справочников (Ingredients, Tags)
    - B2. CRUD для Recipes (header + instructions)
    - B3. Логика связывания Recipe + Ingredients
    - B4. Реализация фильтра «поиск по ингредиентам»
    - B5. Валидация входных данных

- Эпик C. Планирование и Покупки
  - Фичи: Meal Plan, Shopping List Logic
  - Задачи:
    - C1. CRUD для MealPlan (проверка дат, типов приема пищи)
    - C2. CRUD для ShoppingList
    - C3. Логика генерации списка покупок из MealPlan (агрегация количества)
    - C4. Отметка купленных товаров

- Эпик D. UI / Интерфейс пользователя
  - Фичи: Каталог, Календарь, Чек-лист
  - Задачи:
    - D1. Главная страница (поиск и фильтры)
    - D2. Страница создания рецепта (динамические поля ингредиентов)
    - D3. Страница деталей рецепта
    - D4. Календарь питания (drag & drop или выбор слота)
    - D5. Интерактивный список покупок

- Эпик E. Безопасность и роли
  - Фичи: Auth, Permissions
  - Задачи:
    - E1. Регистрация/Логин (JWT)
    - E2. Защита роутов (User видит только свои планы)
    - E3. Роль Admin (редактирование глобальных справочников)

- Эпик F. Окружение и CI/CD
  - Фичи: Docker, Tests
  - Задачи:
    - F1. Dockerfile & docker-compose
    - F2. CI pipeline (lint, build)
    - F3. Unit-тесты (сервисы планирования)
    - F4. Интеграционные тесты API

## Приоритеты (MVP)

1. A1-A3 — БД и сиды (без ингредиентов приложение пустое)
2. B1-B4 — Поиск и создание рецептов
3. E1-E2 — Авторизация (нужна для сохранения планов)
4. C1-C3 — Планирование и список покупок (основная ценность)
5. D1-D5 — Frontend
