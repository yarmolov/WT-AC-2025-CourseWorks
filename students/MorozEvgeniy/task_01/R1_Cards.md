# Вариант 18 — Карточки (обязательные поля и валидации)

## 1. Форма аутентификации

- email: string, format: email → ошибка: «Некорректный email»
- password: string, min 6 chars → ошибка: «Слишком короткий пароль»

## 2. Карточка рецепта (Recipe)

- title: string, не пустое, 3-100 символов → ошибка: «Введите название рецепта»
- description: string, max 500 символов
- prep_time: integer, > 0 → ошибка: «Время должно быть больше 0»
- servings: integer, > 0 → ошибка: «Укажите количество порций»
- ingredients: Array, не пустой → ошибка: «Добавьте хотя бы один ингредиент»
- steps: string/text, не пустое → ошибка: «Опишите процесс приготовления»

## 3. Карточка ингредиента в рецепте (RecipeIngredient)

- ingredient_id: UUID, из справочника → ошибка: «Выберите продукт»
- amount: number, > 0 → ошибка: «Количество должно быть положительным»
- unit: string (гр, кг, шт, мл) → ошибка: «Выберите единицу измерения»

## 4. Карточка плана питания (MealPlan)

- date: date, формат YYYY-MM-DD
- meal_type: enum [breakfast, lunch, dinner, snack]
- recipe_id: UUID, exists → ошибка: «Рецепт не найден»

## 5. Карточка товара в покупках (ShoppingItem)

- name: string (если нет ID ингредиента) → ошибка: «Название товара обязательно»
- amount: number, min 0
- is_bought: boolean, default: false

## Пользовательские истории (User Stories)

## US-01: Поиск по холодильнику

**Как** голодный пользователь
**Я хочу** найти рецепты, используя список продуктов, которые у меня есть
**Чтобы** не идти в магазин и использовать остатки еды

**Критерии приёмки (AC):**
- Пользователь выбирает ингредиенты (напр. Яйца, Томаты)
- Выдача содержит рецепты, где эти ингредиенты основные
- Сортировка по релевантности (максимальное совпадение)

## US-02: Создание рецепта

**Как** кулинар
**Я хочу** опубликовать свой рецепт с ингредиентами
**Чтобы** сохранить его для себя и поделиться с другими

**Критерии приёмки (AC):**
- Можно добавить фото (ссылка или файл)
- Ингредиенты выбираются из выпадающего списка
- Инструкция сохраняется с форматированием (или пошагово)

## US-03: Планирование недели

**Как** пользователь
**Я хочу** раскидать рецепты по дням недели
**Чтобы** знать, что готовить завтра

**Критерии приёмки (AC):**
- Визуальное отображение календаря
- Один клик для перехода к рецепту из календаря
- Валидация: нельзя добавить на несуществующую дату

## US-04: Генерация списка покупок

**Как** пользователь
**Я хочу** получить список продуктов на основе моего плана питания
**Чтобы** купить всё за один раз

**Критерии приёмки (AC):**
- Система суммирует одинаковые ингредиенты (было 2 яйца на завтрак и 3 в пирог → итог 5 яиц)
- Можно отметить товар как купленный
- Можно добавить свои товары (напр. «Губки для посуды»)

## US-05: Учет диет

**Как** вегетарианец
**Я хочу** видеть значки или фильтровать рецепты по тегам
**Чтобы** случайно не приготовить блюдо с мясом

**Критерии приёмки (AC):**
- При поиске есть чекбокс «Вегетарианское»
- В карточке рецепта отображаются теги
