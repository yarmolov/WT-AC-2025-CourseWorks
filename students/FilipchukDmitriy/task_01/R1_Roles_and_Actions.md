# Вариант 16 — Заметки «Пишем вдвоём» — Роли и действия

Краткое описание

MVP: Notebook, Note, Label, Share, User. UI отображает тетради с заметками, метки и совместный доступ; API: /notebooks, /notes, /labels, /shares. Приёмка (MVP): корректное создание и управление доступами (Share).

Роли и основные действия

- Администратор
  - CRUD для пользователей
  - CRUD для тетрадей (Notebook) и заметок (Note)
  - Просмотр всех данных в системе
  - Управление метками (Label)

- Пользователь
  - Создание и редактирование своих тетрадей и заметок
  - Управление метками для своих заметок
  - Предоставление доступа к тетрадям (Share)
  - Просмотр тетрадей, к которым предоставлен доступ
  - Совместное редактирование заметок (если есть доступ)

Примеры сценариев

- Пользователь создаёт тетрадь, добавляет заметки, присваивает метки → делится тетрадью с другим пользователем → оба могут редактировать заметки.
- Администратор просматривает список пользователей и их активность, может удалять аккаунты.
