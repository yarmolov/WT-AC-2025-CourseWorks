# Вариант 37 — Новости «Без фейков» — Роли и действия

Краткое описание

MVP: Source, Article, Tag, Favorite, Report, User. UI отображает ленту новостей и избранное; API: /sources, /feed, /tags, /favorites, /reports. Приемка (MVP): корректная модерация жалоб.

Роли и основные действия

- Администратор
  - CRUD для пользователей
  - CRUD для источников (Source) и тегов (Tag)
  - Просмотр и модерация жалоб (Report)
  - Просмотр журнала событий

- Модератор
  - Просмотр списка жалоб
  - Рассмотрение и закрытие жалоб
  - Удаление статей по жалобе

- Пользователь
  - Просмотр ленты новостей (Feed)
  - Фильтрация статей по тегам
  - Добавление/удаление статей в избранное
  - Подача жалобы на статью

Примеры сценариев

- Администратор добавляет Source и Tag → система агрегирует статьи → пользователь видит ленту новостей.
- Пользователь находит подозрительную статью, подаёт жалобу → модератор рассматривает и принимает решение.
