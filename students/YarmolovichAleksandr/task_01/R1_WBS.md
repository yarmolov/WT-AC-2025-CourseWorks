# Вариант 38 — WBS (эпики → фичи → задачи)

- Эпик A. Модель данных и миграции

- Фичи: схемы User, Category, Ad, Media, Conversation, Message, Report; миграции
  - Задачи: A1. Описать таблицы и связи (ERD); A2. Написать миграции для всех сущностей; A3. Подготовить демо-данные (пользователи, категории, тестовые объявления).

- Эпик B. Объявления и категории

- Фичи: CRUD категорий; создание/редактирование объявлений; фильтры; медиа
- Задачи: B1. CRUD /categories; B2. CRUD /ads; B3. Реализовать фильтры: категория, поиск, цена;
B4. Загрузка и привязка медиа (/media, /ads/{id}/media)

- Эпик C. Чаты

- Фичи: создание чата, обмен сообщениями, список диалогов
- Задачи: C1. Реализовать /conversations (создание, список); C2. Реализовать /messages (отправка/получение);  C3. WebSocket для новых сообщений (бонус).

- Эпик D. Жалобы и модерация

- Фичи: система жалоб; модерация; статусы; управление объявлениями
- Задачи: D1. Реализовать /reports (создать, список, смена статуса); D2. Логика блокировки объявления;
D3. Панель модератора с фильтрами; D4. Тесты корректности модерации.

- Эпик E. Аутентификация и роли

- Фичи: логин/регистрация; роли user/moderator/admin; права доступа
- Задачи: E1. JWT аутентификация; E2. Реализовать RBAC (роли и их права); E3. Тесты на проверку прав;
 E4. Документация матрицы ролей.

- Эпик F. UI / интерфейс пользователя

- Фичи: лента объявлений; поиск и фильтры; чат; профиль; модераторка
- Задачи: F1. Лента объявлений и поиск; F2. Страница объявления; F3. Создание/редактирование объявления; F4. UI чатов; F5. UI жалоб и модерации.

Эпик G. Бонусы и продвинутая функциональность

- Фичи: фейковые платежи; уведомления
- Задачи: G1. Система уведомлений (WS или push); G2. Платежи-заглушки (опционально).
