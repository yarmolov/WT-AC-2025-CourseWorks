# Вариант 8 — Доска объявлений «Бери, пока горячее» — Маршруты (user flows)

1. Идентификация (вход в систему)
    - Пользователь: логин → переход в ленту объявлений
    - Администратор/Модератор: логин → админ-панель (пользователи, жалобы, категории)

2. Подготовка данных (Администратор)

   - Создать категории
   - Управлять пользователями (назначение модераторов)
   - Настроить общие параметры площадки

3. Работа с объявлениями (Пользователь / Модератор / Администратор)

   - Создать объявление (title, description, price, category)
   - Загрузить фотографии
   - Редактировать/удалить объявление (пользователь — только свои)
   - Модератор/Администратор может заблокировать объявление

4. Взаимодействие между пользователями (чат)

   - Покупатель открывает объявление → нажимает “Написать”
   - Создаётся Conversation
   - Пользователь отправляет сообщения
   - Продавец отвечает, продолжается диалог

5. Просмотр пользователем (Пользователь)

   - Просмотр списка объявлений (лента + фильтры)
   - Просмотр категорий
   - Просмотр карточки объявления
   - Просмотр чатов и сообщений

6. Жалобы и модерация

   - Пользователь подаёт жалобу на объявление
   - Модератор получает список новых жалоб
   - Проверяет объявление и принимает решение: оставить / заблокировать / отклонить
   - Статус жалобы обновляется

7. История / Журнал (Администратор)

   - Просмотр активности пользователей
   - Просмотр статистики жалоб
   - Просмотр действий модераторов
   - Логирование изменений объявлений
