# Вариант 02 — Бронь аудиторий «Не занято?» — Маршруты (user flows)

1. Идентификация (вход в систему)

   - Студент: логин → дашборд с календарём аудиторий и своими бронированиями
   - Преподаватель: логин → дашборд с календарём и функциями бронирования
   - Администратор: логин → админ-панель с управлением аудиториями и пользователями

2. Подготовка данных (Администратор)

   - Заполнить список пользователей
   - Создать аудитории (Room)
   - Назначить роли пользователям (студент/преподаватель/админ)

3. Работа с аудиториями (Администратор)

   - Добавить/редактировать/удалить Room
   - Установить параметры аудитории (вместимость, оборудование)
   - Просмотр статистики использования аудиторий

4. Бронирование аудитории (Студент/Преподаватель)

   - Просмотр календаря доступных аудиторий
   - Выбор аудитории, даты и времени
   - Создание бронирования с указанием цели
   - Система проверяет конфликты и подтверждает бронь

5. Просмотр расписания (Все роли)

   - Просмотр календаря по аудиториям
   - Фильтрация по дате, времени, типу аудитории
   - Просмотр деталей бронирования (кто, когда, для чего)

6. Управление бронированиями (Пользователь/Администратор)

   - Просмотр своих бронирований
   - Отмена бронирования
   - Перенос бронирования на другое время
   - Система проверяет возможность переноса

7. Обработка конфликтов

   - Система автоматически предотвращает пересечения бронирований
   - Отображение занятых слотов времени
   - Предложение альтернативных вариантов

8. Отчёты / История

   - Просмотр истории бронирований по аудитории
   - Просмотр личной истории бронирований
   - Экспорт расписания в iCal (бонус)
