# Вариант 02 — WBS (эпики → фичи → задачи)

- Эпик A. Модель данных и миграции
  - Фичи: схемы User/Room/Booking/Role; миграции
  - Задачи: A1. Описать таблицы и связи; A2. Написать миграции; A3. Подготовить демо‑данные

- Эпик B. API и интеграция данных
  - Фичи: REST API (/rooms, /bookings, /schedule, /users); валидация конфликтов
  - Задачи: B1. Реализовать endpoints CRUD; B2. Реализовать логику проверки конфликтов; B3. Добавить пагинацию и фильтры

- Эпик C. Логика бронирования и конфликтов
  - Фичи: проверка занятости, предотвращение пересечений, отмена/перенос
  - Задачи: C1. Реализовать engine проверки конфликтов; C2. Тесты на пересечения времени; C3. UI для предупреждений о конфликтах

- Эпик D. UI / календарь
  - Фичи: календарь аудиторий, форма бронирования, список своих бронирований
  - Задачи: D1. Компонент календаря (grid view); D2. Форма создания бронирования; D3. Список бронирований с фильтрами

- Эпик E. Безопасность и роли
  - Фичи: аутентификация, авторизация, права по ролям (студент/преподаватель/админ)
  - Задачи: E1. Реализовать RBAC; E2. Тесты доступа по ролям; E3. Документация прав

- Эпик F. Окружение и CI/CD
  - Фичи: Docker, CI pipeline, тесты, развёртывание (Kubernetes бонус)
  - Задачи: F1. Dockerfile для front и back; F2. Github Actions/CI; F3. Подготовить инструкции для демо
