# Вариант 1 — Task Manager — Роли и действия

Краткое описание

MVP: Task, User, Comment, Attachment, Notification. UI отображает список задач, доску Kanban и детали задачи; API: /tasks, /users, /comments, /attachments. Приёмка (MVP): создание задачи, назначение исполнителя, изменение статуса.

Роли и основные действия

- Администратор
  - CRUD для пользователей
  - CRUD для задач (Task) любых пользователей
  - Назначение ролей пользователям
  - Просмотр всех задач и отчётов

- Менеджер
  - Создание и редактирование задач
  - Назначение задач пользователям
  - Просмотр задач своей команды
  - Закрытие и проверка задач

- Пользователь
  - Просмотр назначенных задач
  - Изменение статуса своих задач
  - Добавление комментариев к задачам
  - Загрузка вложений к задачам

Примеры сценариев

- Менеджер создаёт задачу и назначает исполнителя → система создаёт уведомление → пользователь меняет статус задачи.
- Администратор просматривает отчёт по выполненным задачам за месяц, фильтрует по пользователям.
