# Вариант 27 — Pomodoro «Томатная продуктивность» — Роли и действия

Краткое описание

MVP: Task, Session, Tag, User. UI отображает таймер, задачи и статистику; API: /tasks, /sessions, /reports. Приёмка (MVP): корректная работа таймера с уведомлениями при завершении сессии.

Роли и основные действия

- Администратор
  - CRUD для пользователей
  - Просмотр статистики по всем пользователям
  - Просмотр всех задач и сессий
  - Управление глобальными настройками (длительность Pomodoro, перерывов)

- Пользователь
  - CRUD для своих задач (Task)
  - CRUD для своих тегов (Tag)
  - Запуск, пауза, возобновление, завершение Pomodoro-сессий (Session)
  - Просмотр своей статистики и отчётов
  - Настройка уведомлений (push/email/sound)
  - Получение уведомлений о завершении сессии
  - Экспорт данных за период

Примеры сценариев

- Пользователь создаёт задачу, запускает таймер на 25 минут → система отсчитывает время → при завершении создаётся Session и отправляется уведомление.
- Пользователь просматривает статистику за неделю: количество сессий, общее время работы, распределение по тегам.
