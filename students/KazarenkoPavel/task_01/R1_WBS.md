# Вариант 27 — WBS (эпики → фичи → задачи)

- Эпик A. Модель данных и миграции
  - Фичи: схемы Task/Tag/Session/User; миграции
  - Задачи: A1. Описать таблицы и связи; A2. Написать миграции; A3. Подготовить демо-данные

- Эпик B. API и бизнес-логика
  - Фичи: REST API (/tasks, /sessions, /reports); валидация данных
  - Задачи: B1. Реализовать endpoints CRUD для Task/Tag; B2. Реализовать управление Session; B3. Добавить пагинацию и фильтры

- Эпик C. Логика таймера и уведомления
  - Фичи: таймер Pomodoro (клиентский с серверной синхронизацией), уведомления о завершении сессии
  - Задачи: C1. Реализовать клиентский таймер с отображением обратного отсчёта; C2. Синхронизация состояния сессии с сервером (start/pause/resume/complete); C3. Интеграция уведомлений (push/sound через NotificationSettings); C4. Тесты на корректность работы таймера и пауз
  - Примечание: таймер работает на клиенте; сервер хранит start_time, paused_at, total_paused_seconds для восстановления состояния при перезагрузке страницы

- Эпик D. UI / интерфейс
  - Фичи: страница задач, таймер, статистика
  - Задачи: D1. Компонент таймера с визуализацией; D2. Страница списка задач; D3. Страница статистики с графиками

- Эпик E. Отчёты и статистика
  - Фичи: дневная/недельная/месячная статистика, экспорт данных
  - Задачи: E1. Реализовать endpoints отчётов; E2. Визуализация графиков; E3. Экспорт в CSV/JSON

- Эпик F. Безопасность и роли
  - Фичи: аутентификация, авторизация, RBAC, проверка владельца ресурсов
  - Задачи: F1. Реализовать JWT auth; F2. Middleware checkOwnership для Task/Tag/Session; F3. Тесты доступа; F4. Документация прав
  - Примечание: роль admin назначается через seed-скрипт или SQL (UPDATE users SET role='admin' WHERE email='...')
