# Вариант 35 — WBS (эпики → фичи → задачи)

- Эпик A. Модель данных и миграции
  - Фичи: схемы Trip/Stop/Note/Expense/User/TripParticipant; миграции
  - Задачи: A1. Описать таблицы и связи; A2. Написать миграции; A3. Подготовить демо‑данные

- Эпик B. API и интеграция данных
  - Фичи: REST API (/trips, /stops, /notes, /expenses); WebSocket (бонус)
  - Задачи: B1. Реализовать endpoints CRUD; B2. Реализовать шаринг поездок; B3. Добавить пагинацию и фильтры

- Эпик C. Совместная работа и шаринг
  - Фичи: приглашение участников, права доступа, уведомления
  - Задачи: C1. Реализовать механизм добавления участников; C2. Тесты на права доступа; C3. UI для управления участниками

- Эпик D. UI / интерфейс
  - Фичи: дашборд с поездками, карточка поездки, карта маршрута
  - Задачи: D1. Компонент списка поездок; D2. Страница деталей поездки; D3. Интеграция карты для отображения остановок

- Эпик E. Управление бюджетом
  - Фичи: добавление расходов, отслеживание бюджета, категории
  - Задачи: E1. Реализовать CRUD для Expense; E2. Расчёт остатка бюджета; E3. UI для добавления расходов

- Эпик F. Безопасность и роли
  - Фичи: аутентификация, авторизация, scope доступа
  - Задачи: F1. Реализовать RBAC; F2. Тесты доступа; F3. Документация прав

- Эпик G. Окружение и CI/CD
  - Фичи: Docker, CI pipeline, тесты
  - Задачи: G1. Dockerfile; G2. Github Actions/CI; G3. Подготовить инструкции для демо
