# Вариант 35 — Поездки «Поехали!» — Роли и действия

Краткое описание

MVP: Trip, Stop, Note, Expense, User. UI отображает маршруты, места, даты, бюджет и заметки; API: /trips, /stops, /notes, /budget. Приёмка (MVP): корректное создание и шаринг поездки между пользователями.

Роли и основные действия

- Администратор
  - CRUD для пользователей
  - Просмотр всех поездок
  - Модерация контента
  - Просмотр статистики системы

- Пользователь
  - CRUD для своих поездок (Trip)
  - CRUD для остановок (Stop) в поездке
  - CRUD для заметок (Note)
  - CRUD для расходов (Expense)
  - Приглашение других пользователей в поездку (шаринг)
  - Просмотр совместных поездок
  - Просмотр карты маршрута

Примеры сценариев

- Пользователь создаёт Trip, добавляет остановки Stop с датами и адресами, устанавливает бюджет → система сохраняет данные → пользователь приглашает друзей через шаринг.
- Пользователь добавляет Expense к поездке, система пересчитывает оставшийся бюджет и отображает на дашборде.
