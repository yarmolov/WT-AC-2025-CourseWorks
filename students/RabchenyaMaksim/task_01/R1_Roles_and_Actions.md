# Вариант 31 — Городские жалобы «Чиним город» — Роли и действия

Краткое описание

MVP: Report, Category, Comment, Location, User. UI отображает карту с заявками и форму создания жалоб; API: /reports, /categories, /comments, /moderation. Приёмка (MVP): отображение заявок на карте с фильтрацией по статусам и категориям.

Роли и основные действия

- Администратор
  - CRUD для пользователей и модераторов
  - CRUD для категорий заявок
  - Просмотр всех заявок и комментариев
  - Модерация любых заявок
  - Доступ к полной статистике и отчетам

- Модератор
  - Просмотр всех заявок и комментариев
  - Модерация заявок (одобрение/отклонение)
  - Управление статусами заявок
  - Удаление некорректных комментариев
  - Доступ к базовой статистике

- Пользователь
  - Создание новых заявок (Report)
  - Просмотр публичных заявок и комментариев
  - Добавление комментариев к заявкам
  - Редактирование своих заявок (ограниченное время)
  - Просмотр карты с заявками

Примеры сценариев

- Пользователь создает заявку о яме на дороге → система определяет местоположение → заявка появляется на карте со статусом "новая".
- Модератор проверяет новые заявки → одобряет корректные → статус меняется на "в работе".
- Администратор анализирует статистику по категориям заявок → создает новые категории для частых проблем.
