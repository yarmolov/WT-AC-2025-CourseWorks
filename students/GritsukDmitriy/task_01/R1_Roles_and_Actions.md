# Вариант 24 — Roadmaps «Как стать джуном» — Роли и действия

Краткое описание

MVP: Roadmap, Step, Resource, Progress, User. UI отображает дорожные карты с шагами и трекинг прогресса; API: /roadmaps, /steps, /progress, /resources. Приёмка (MVP): корректный трекинг прогресса пользователя по шагам дорожной карты.

Роли и основные действия

- Администратор
  - CRUD для пользователей
  - CRUD для дорожных карт (Roadmap) — включая управление полями category, difficulty, is_published
  - CRUD для шагов (Step)
  - CRUD для ресурсов (Resource)
  - Просмотр прогресса всех пользователей (только просмотр, без редактирования)
  - Просмотр и редактирование своего профиля

- Пользователь
  - Просмотр списка опубликованных дорожных карт (is_published=true)
  - Фильтрация roadmaps по категории и сложности
  - Просмотр шагов и ресурсов (только из опубликованных roadmaps)
  - Отметка шагов как выполненных (Progress) — только свой прогресс
  - Просмотр своего прогресса
  - Просмотр и редактирование своего профиля (username, email, password, но не role)

Примеры сценариев

- Администратор создаёт Roadmap «Frontend Developer», задаёт категорию «Frontend», сложность «beginner», добавляет шаги (HTML, CSS, JS...), прикрепляет ресурсы, публикует roadmap (`is_published=true`) → пользователь видит карту и начинает проходить.
- Пользователь регистрируется с email и паролем → автоматически получает роль `user` → может фильтровать roadmaps по категории «Backend» и сложности «intermediate» → выбирает подходящую карту.
- Пользователь открывает Roadmap, изучает шаг, отмечает его как выполненный → система обновляет Progress (связывая `user_id` из JWT с `step_id`) и показывает процент завершения.
- Пользователь редактирует свой профиль: меняет email и пароль → система валидирует данные (email валидный, пароль минимум 8 символов) → сохраняет изменения, но не позволяет менять `role`.
