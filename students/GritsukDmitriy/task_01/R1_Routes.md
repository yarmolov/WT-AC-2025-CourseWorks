# Вариант 24 — Roadmaps «Как стать джуном» — Маршруты (user flows)

1. Идентификация (вход в систему)

   - Пользователь: логин → дашборд с дорожными картами и прогрессом
   - Администратор: логин → админ-панель

2. Подготовка данных (Администратор)

   - Заполнить список пользователей
   - Создать дорожные карты (Roadmap)
   - Создать шаги (Step)
   - Создать ресурсы (Resource)
   - Настроить связи Roadmap ↔ Step ↔ Resource

3. Работа с дорожными картами (Администратор)

   - Добавить/редактировать/удалить Roadmap
   - Добавить/редактировать/удалить Step
   - Добавить/редактировать/удалить Resource

4. Изучение дорожной карты (Пользователь)

   - Просмотр списка доступных Roadmap
   - Выбор Roadmap для прохождения
   - Просмотр шагов (Step) внутри Roadmap
   - Просмотр ресурсов (Resource) для каждого шага

5. Трекинг прогресса (Пользователь)

   - Отметить шаг как выполненный
   - Просмотр своего прогресса (Progress) по Roadmap
   - Просмотр общего процента выполнения

6. Просмотр статистики (Администратор/Пользователь)

   - Администратор: просмотр прогресса всех пользователей
   - Пользователь: просмотр только своего прогресса

7. Поиск и фильтрация

   - Фильтрация Roadmap по категории (Frontend, Backend, DevOps, и т.д.)
   - Фильтрация по уровню сложности (beginner, intermediate, advanced)
   - Отображение только опубликованных roadmaps для пользователей

8. Редактирование профиля (Пользователь)

   - Просмотр своего профиля (username, email)
   - Редактирование email
   - Изменение пароля
   - Сохранение изменений
