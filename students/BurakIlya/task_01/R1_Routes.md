# Вариант 43 — Волонтёры «Помощь рядом» — Маршруты (user flows)

1. Идентификация (вход в систему)

   - Пользователь (role: user): логин → дашборд с запросами помощи
   - Пользователь с VolunteerProfile: логин → панель доступных заявок + свой дашборд
   - Администратор (role: admin): логин → админ-панель

2. Подготовка данных (Администратор)

   - Заполнить список пользователей
   - Создать категории помощи (Category)
   - Создать профили волонтёров (VolunteerProfile)
   - Настроить локации и зоны обслуживания

3. Работа с категориями и профилями (Администратор)

   - Добавить/редактировать/удалить Category
   - Управление профилями волонтёров
   - Модерация заявок и отзывов

4. Создание запроса помощи (Пользователь)

   - Создать новый запрос помощи (HelpRequest)
   - Указать категорию, описание, локацию
   - Система сохраняет запрос со статусом "new"

5. Поиск и просмотр запросов (Волонтёр)

   - Просмотр списка активных запросов помощи
   - Фильтрация по категории и локации
   - Просмотр деталей запроса (описание, расстояние)

6. Назначение на запрос (User с VolunteerProfile / Администратор)

   - User с VolunteerProfile откликается на запрос (проверка: request.status = 'new')
   - Администратор может назначить любого волонтёра
   - Создаётся Assignment со статусом "assigned"
   - Запрос помощи переходит в статус "assigned"

7. Работа с заявкой (Волонтёр и Пользователь)

   - Чат между волонтёром и пользователем (заглушка)
   - Волонтёр отмечает начало/завершение помощи
   - Статус Assignment меняется на "in_progress" → "completed"

8. Оставление отзыва (Пользователь)

   - После завершения помощи пользователь может оставить Review
   - Оценка и комментарий к волонтёру
   - Отзыв отображается в профиле волонтёра

9. История и отчёты

   - Просмотр истории запросов и выполненных заданий
   - Статистика для волонтёров (количество помощи, рейтинг)
   - Экспорт данных для анализа
