## Введение

## 1. Постановка задачи
- Исходные данные (форматы файлов, текстовое описание схемы данных)
- Функциональные требования (MVP + опции)
- Нефункциональные требования (производительность, безопасность)
- Ограничения и допущения
- Чеклист приёмки MVP (технический чеклист)

## 2. Разработка алгоритмов
- Описание сущностей и связей (текст/таблицы — без графики)
- Алгоритм формирования ленты (псевдокод)
- Алгоритмы ключевых операций (псевдокод/описание)
- Транзакции и согласованность данных
- Схема аутентификации (текстовое описание JWT flow)

## 3. Разработка программы (реализация)
- Архитектура системы (текстовое описание: client, server, DB, storage)
- Выбранный техстек и обоснование
- Описание модулей сервера (routes/controllers/services/repositories)
- Описание фронтенда (страницы, ключевые компоненты, управление состоянием)
- API‑спецификация (таблично: endpoint → метод → request → response → ошибки)
- Модель данных (таблицы с полями и типами; индексация указывается текстом)
- Конфигурация окружения и инструкция по запуску (Docker Compose команды)
- План деплоя (Docker; Kubernetes опционально — текстовое описание)

## 4. Тестирование
- План тестирования: unit, integration, e2e
- Список тестовых сценариев (регистрация, логин, создание поста, лента, лайк, модерация)
- Критерий прохождения тестов: pass/fail на каждый тест
- Отчёт о тестах (заполняется при реализации)
