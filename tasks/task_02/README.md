# Задание #2: Финальный исходный код курсового проекта (src)

Цель: сдать **полный (финальный) исходный код** вашего курсового проекта.

Сдать в PR в папку `students/<NameLatin>/task_02/`:

- `src/` — исходники проекта (полная реализация).
- `README.md` — инструкция по запуску.

## Что значит «финальный/полный исходный код»

- В `src/` должен быть **весь код приложения**, который вы сдаёте как результат курсового проекта.
- Это **не каркас** и не «черновик»: должны быть реализованы основные функции вашего варианта.
- Проект должен собираться/запускаться локально по инструкции.

## Минимальные требования

- В `students/<NameLatin>/task_02/src/` — полный набор исходников.
- В `students/<NameLatin>/task_02/README.md` — понятная инструкция:
	- как установить зависимости (если есть),
	- как запустить dev/preview,
	- какая версия Node (если критично),
	- где посмотреть результат (URL/порт).
- Если используете сборщик (Vite/webpack и т.п.), конфиги и `package.json` должны лежать **внутри** `students/<NameLatin>/task_02/`.
- Не добавляйте в PR файлы вне `students/<NameLatin>/...`.

## Рекомендуемая структура (пример)

- `src/` — исходники
- `public/` или `assets/` — статика (если нужно)
- `package.json` — зависимости и скрипты (если используется Node)

## Приёмка

- Код является «полной сдачей»: присутствуют ключевые экраны/сценарии и базовая бизнес‑логика.
- Проект реально запускается локально по инструкции из `README.md`.
